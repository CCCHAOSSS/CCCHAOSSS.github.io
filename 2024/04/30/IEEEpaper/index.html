IJCNN 论文提交过程

前情提要：
在投稿快ddl的时候看提交教程发现上传pdf时候会出现以下问题，然后再没有尝试提交就开始根据教程解决，最后发现，其实直接提交就可以！！

然后。。几个月后提交最终版本时候终于还是遇到了这些问题，遂记录下来！！太搞了！！

首先是 font XX is not embedded

Upload failed: The font Arial,Italic is not embedded in the file.
这个确实是由于字体未嵌入引起的。插入的pdf图片是在ppt中画出来的，在导出为pdf时候需要在选项中选择符合PDF/A。

然后是Font ArialMT is of type Type 3

fonttype: Upload failed: Font ArialMT is of type Type 3.

在我按照上面操作重新生成pdf并编译得到论文最终版上传后，又出现了这个提示。这下连在哪的字体有问题都不知道了（吐血）。

经过我的一步一步排查（就是注释掉图片一个一个看），发现问题出在用origin画的图的字体上。在经过多方检索后，发现只要在导出pdf时候进行如下操作：

  • File -> Export Graphs -> Open Dialog -> 弹出导出图片对话框。
  • Image Type 选择PDF格式，然后一直拖下来，找到Image Setting-> options->Fonts -> 选择 “Adobe Type1”+”Use Built-in Fonts” 即可。

但是，我提交后仍旧报错了！！！！
再经过我呕心沥血的检查后，发现是$\Delta$这个符号的问题……

我真的……被整的没脾气了，所以记住，origin里面：

不要使用输入法打出来的特殊字符！

不要使用输入法打出来的特殊字符！

不要使用输入法打出来的特殊字符！

使用origin提供的输入特殊字符的方法，网上都有教程我就不说了，很简单的，也是可以直接键盘输入。

最后是created by ‘TeX’ instead

notcertified: Upload failed: The PDF file has not been certified by PDF eXpress; created by 'TeX' instead.

这个就很简单了，从overleaf导出的PDF要经过eXpress审查，然后上传审查完后提供的pdf就万事大吉了！

历时5个月的论文终于从开始写到最终版本提交结束了~！虽然只是C会，但是也算给研究生生涯画上句号了。唯一的遗憾就是没能亲自去日本参会

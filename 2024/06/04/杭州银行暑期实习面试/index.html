  • jdk1.7和1.8的HashMap有什么区别
    （答了一个扩容时候链表插入的方式不一样，然后想不起来了=。=）
    1. 数据结构的优化

    Java 1.8引入红黑树，解决大量哈希冲突时链表性能下降的问题。当链表长度超过8时，链表转换为红黑树，使得查询、插入、删除操作的平均时间复杂度从 O(n) 降低到 O(log n)。

    1. 扩容机制优化
      Java 1.7：扩容时，所有元素都需要重新计算哈希值并移动到新的桶位置。

    Java 1.8：扩容时，通过简单的位运算（hash & (newCapacity - 1)）判断元素是保留在原桶位置还是移动到新的桶位置，减少了重新哈希计算的复杂度。

    1. 并发问题改进
      Java 1.7：在多线程环境下，如果多个线程同时对 HashMap 进行插入操作，可能导致链表形成环，最终导致死循环。
      Java 1.8：通过引入 TreeNode 和对链表的优化，减少了链表形成环的可能性，降低了死循环的风险。但仍需注意，HashMap 本身不是线程安全的，多线程环境下应使用 ConcurrentHashMap。

总结：总结
Java 1.7 的 HashMap数据结构简单，链表用于处理哈希冲突；扩容时所有元素重新计算哈希；多线程环境下扩容可能导致死循环。

Java 1.8 的 HashMap引入了红黑树优化哈希冲突处理，提高性能；扩容机制优化，减少重新哈希的开销；并发问题有所改善，但仍建议在多线程环境下使用 ConcurrentHashMap。

  • MySQL索引失效情况

老生常谈的问题了。。答得也不是很好，比较浅。

  1. 使用> < = != 时候就可能会失效
  2. 使用or条件时，如果or的每个条件都没有索引，则索引会失效
  3. 隐式类型转换。如果查询中的列类型和条件类型不匹配，MySQL 会进行隐式类型转换，导致索引失效。
  4. 以%开头的like模糊查询。
  5. 查询条件不匹配索引顺序，可能会导致索引失效。如创建索引CREATE INDEX idx_name_age ON users(username, age);，但SELECT * FROM users WHERE age = 25 AND username = 'john';

5.接口、抽象类普通类的区别

从实现方面和设计层面都说了一些，答了一半，实现上有点忘了=。= ，而且大多在答抽象类和接口的区别，对普通类说得比较少。

接口和抽象类语法层面上：

  • 抽象类中可以有方法的实现，可以有抽象或非抽象方法；接口中的方法只能是抽象方法（不能有实现）。
  • 抽象类中的成员变量访问权限可以是各种各样的，接口中只能是public staic final。
  • 抽象类中可以有静态代码块、静态方法static，接口在1.8前都不能有这些，1.8后可以有静态方法。
  • 抽象类是类，只能被单继承，而一个类可以实现多个接口。

抽象类和普通类区别：

  • 抽象类不能被实例化，抽象类存在的目的是作为其他类的基类；普通类是可以被实例化的，包含具体的方法和属性，可以直接用。
  • 抽象类可以包含抽象方法也可以包含具体的方法，抽象方法只有声明没有实现，子类必须实现这些抽象方法；普通类是用于定义和实现具体的对象行为，可以直接使用和实例化。

6.springboot和spring的区别

也是比较宽泛地答了一些。具体的：

Spring是一个全面的功能丰富的java框架，提供了依赖注入（DI）、面向切面编程（AOP）、事务管理、数据访问、WebMVC等。Spring需要手动配置和管理各种组件。

而SpringBoot是基于Spring框架，简化Spring应用的开发，通过约定大于配置、内嵌服务器（Tomcat等）和自动配置，极大减少开发Spring项目所需工作和时间。

常见区别：

区别 spring springboot
配置 需要大量XML配置或java配置类 提供自动配置功能
启动 显示定义和配置服务器 内置了嵌入式服务器(Tomcat、netty等)
依赖 需要手动配置和管理各个组件 提供了Starters依赖，可以快速添加各种功能和库，简化了依赖管理

具体区别

1.项目初始化
  • spring
    • 手动配置各种元数据，包括XML文件、Java配置类（就是定义配置参数的一个类）、注解等。开发者需要显示指定各种组件，例如数据源、事务管理器等。
  • springboot
    • 采用约定大于配置，通常只需要创建一个包含@SpringBootApplication注解的主类，SpringBoot 会从主类所在的包开始扫描并加载组件，并自动配置您的应用。同时，它还会加载外部配置文件中的属性

    • 会根据添加的依赖项，例如spring-boot-start-web这种。

    • 组件扫描，并加载所有标记为 @Component、@Service、@Controller、 @RestController等的类，并将它们作为 Spring 容器中的 bean。

2.依赖管理
  • spring
    • 要手动显示地配置例如spring-core，spring-web等的依赖（可以直接记spring那些核心模块），引入相应的jar包，并确保这些包的兼容性。
  • springboot
    • 引入starter的概念，这是一组相关的库和依赖的集合。只需要引入相应的starter，springboot就会自动配置

    • 例如，如果你想构建一个Web应用，只需添加spring-boot-starter-web依赖

3.配置
  • spring
    • 通常需要在XML文件或者配置类中配置相关的依赖，手动指定位置，需要写很多东西
  • springboot
    • 在application.yml中配置就行，并且可以配置多个版本。如开发版、生产版（？）
4.内嵌服务器
  • spring
    • 需要手动配置servlet服务器
  • springboot
    • 在starter-web依赖中提供了内嵌的服务器，比如Tomcat、Jetty等。

LRU缓存

hot100—146

请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。
实现 LRUCache 类：LRUCache(int capacity) 以正整数作为容量 capacity 初始化 LRU 缓存

  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；如果不存在，则向缓存中插入该组 key-value 。
  • 如果插入操作导致关键字数量超过 capacity ，则应该逐出最久未使用的关键字。
  • 函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。

话不多说，直接上代码。需要注意的一点是，基础操作是增加节点和删除节点，所以优先把增删写了(并且增加节点的操作只会在头节点进行)，后面的就可以直接调用。并且，需要先自己定义双向链表节点。

写的过程需要注意的点：

  • 当put时，若判断node==null后，要同时插入哈希表（直接调用map的put方法）和链表（用addToHead）
  • 哈希表中key是对应插入的key，value是链表节点DLinkedNode而不是直接key对应的value。

实现前缀树

hot100—208

Trie（发音类似 “try”）或者说 前缀树 是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。这一数据结构有相当多的应用情景，例如自动补完和拼写检查。

  • Trie() 初始化前缀树对象。
  • void insert(String word) 向前缀树中插入字符串 word 。
  • boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false 。
  • boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀之一为 prefix，返回 true；否则，返回false 。

首先要知道什么是前缀树：

一颗有根树，每个节点包含：

  • 指向子节点的指针数组children。
  • 布尔字段idEnd，表示该节点是否为字符串的结尾。

在这道题中，主要先写insert和startPrefix思路都是很像的！

搜索二维矩阵

hot100-240

编写一个高效的算法来搜索 m x n 矩阵 matrix 中的一个目标值 target 。该矩阵具有以下特性：

  • 每行的元素从左到右升序排列。
  • 每列的元素从上到下升序排列。

第一直觉：逐行二分查找。能AC

看题解说还有Z形状查找：

  • matrix[x][y]<target时候，对行操作（+1）
  • matrix[x][y]>target时候，对列操作（-1）

对于二分搜索，时间复杂度是O(m*logn)

而Z形搜索是O(m+n),y 最多能被减少n次，x最多能被增加m次，总搜索次数为m+n。

旋转图像

hot100—48

给定一个 n × n 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。
你必须在 原地 旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要 使用另一个矩阵来旋转图像。

这题官方题解很。。。难看，但是可以记住的一点是，旋转后的矩阵，在[i,j]处的值会落到[j,n-i-1]处。（下标都从0开始）
然后看到一个解法比较适合：先进行矩阵转置，再将每一行进行翻转就可以。需要注意的是：在进行转置操作时候第二个for循环的下标不是从0开始（否则又转回来了）

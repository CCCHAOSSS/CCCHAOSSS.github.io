接口和抽象类内部方法的访问权限

对抽象类来说，抽象类内部方法都是抽象方法abstract，但是抽象类中方法默认访问权限与普通类相同。如果没有指定访问修饰符，则默认为包内可见（即default访问权限）

接口内的方法默认是public，即使没有显示地声明为public。

HTTP和TCP的一些知识

  • TCP是面向连接的，基于字节流的传输协议。

  • HTTP中“垂直越权”：一种安全漏洞，它指的是攻击者试图访问比其权限高的资源或者执行比其权限高的操作。一般来说，垂直越权可能会发生在身份验证和授权过程中。

“垂直越权”是一种安全漏洞，它指的是攻击者试图访问比其权限高的资源或者执行比其权限高的操作。在 HTTP 安全方面，垂直越权可能会发生在身份验证和授权过程中。

在身份验证过程中，如果存在漏洞，攻击者可能会通过欺骗或者绕过身份验证机制，获取到高权限用户的凭证，从而获得对高权限资源的访问权限。

在授权过程中，如果存在漏洞，攻击者可能可以修改请求或者绕过授权机制，来访问其本不应该访问的资源或者执行其本不应该执行的操作。

关于类继承的一些知识点

子类继承父类时候，构造函数遵循以下规则：

  1. 如果子类没有定义构造函数

    • 子类自动继承父类的无参构造函数
      如果父类没有无参构造函数也没有显示定义其他构造函数，那么子类会出现编译错误（这个情况怎么发生啊？？）

  2. 如果子类定义了构造函数

    • 如果子类的构造函数调用了父类的构造函数，则默认调用父类的无参构造函数。
    • 如果父类没有无参构造函数，子类的构造函数必须显式调用父类的其他构造函数（使用 super(…)），否则会出现编译错误。

3. 如果子类定义了与父类相同的构造函数

  • 子类的构造函数会覆盖父类的构造函数。
  • 如果子类构造函数需要调用父类的构造函数，可以使用 super(…) 调用父类的构造函数。

Spring为简化开发采用的策略

  1. 依赖注入（Dependency Injection，DI）和控制反转（Inversion of Control，IoC）：这是 Spring 的核心特性之一。通过依赖注入，Spring 容器负责管理组件之间的依赖关系，开发人员不需要手动创建对象或管理对象之间的关联，从而简化了代码的编写和维护。

  2. 面向接口编程：Spring 鼓励面向接口编程，这有助于降低组件之间的耦合度，提高代码的灵活性和可测试性。

  3. AOP（Aspect-Oriented Programming）：Spring 提供了 AOP 支持，允许将横切关注点（如日志记录、事务管理）从核心业务逻辑中分离出来，以提高代码的模块化和可维护性。

  4. 模板模式：Spring 框架提供了许多模板类，如 JdbcTemplate、RestTemplate 等，简化了与不同资源的交互，如数据库访问、RESTful 服务调用等。

  5. 声明式事务管理：Spring 提供了声明式事务管理的支持，通过使用注解或 XML 配置来管理事务，使得开发者可以将事务逻辑与业务逻辑分离，从而简化了事务管理的实现。

  6. 简化配置：Spring 提供了基于注解的配置，如 @Component、@Autowired 等，以减少 XML 配置的繁琐性，提高了代码的可读性和易维护性。

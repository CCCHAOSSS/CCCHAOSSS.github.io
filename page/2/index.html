  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；如果不存在，则向缓存中插入该组 key-value 。

桶排序

桶排序从整体来看也是一个非比较排序，是计数排序的升级版，其主要是先把序列中的数据划分为一个个的范围，每个范围相当一个桶，将数据放入桶中，每个桶分别进行排序，然后取出拼接即可。计数排序不适用于序列中max-min范围较广或者序列中的数据不是整数的情况，但是桶排序没有这些限制，其主要步骤是将数据放入桶中，桶中的数据自己排序。

我的项目整理

1. 你是如何利用 Redisson 实现分布式锁来保证秒杀功能的正确性的？

  • 在单体模式下，对当前发起请求的userID加synchronized关键字，这里要注意的一点是对userID进行toString操作后的结果加上关键字。然后进入业务逻辑，注意在减库存的service方法上加@Transactional注解。
  • 集群模式下这个方法不行，因为这个同步锁（synchronized）是jvm级别的，在分布式部署后，每台服务器在并发环境下能锁住一个线程。于是使用Redisson。基于Lua脚本，首先判断优惠卷库存是否充足，是则判断用户是否下单，否则扣减优惠卷库存，并将当前UerId存入当前优惠卷的set集合。扣减优惠卷库存部分，也是基于Lua脚本【判断优惠卷是否还有库存，是则将优惠卷id、用户id和订单id存入消息队列，返回订单id】消息队列实现异步下单。
